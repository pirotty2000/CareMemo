# CareMemo リリースビルド作成手順書

このドキュメントでは、CareMemo を「正式版（署名済みリリースビルド）」として書き出し、実機にインストールする手順を説明します。

## 1. 署名鍵（キーストア）の作成
Android アプリを正式なアプリとしてインストールするには、作成者の証明となる「デジタル署名」が必要です。

1.  Android Studio のメニューから **Build > Generate Signed Bundle / APK...** を選択します。
2.  **APK** を選択して **Next** をクリックします。
3.  **Key store path** の下にある **Create new...** をクリックします。
    *   **Key store path**: 鍵ファイルの保存場所を選択します（例: `D:/Users/pirotty.galaxy/Documents/keystore.jks`）。
    *   **Password**: キーストア用のパスワードを設定します。
    *   **Alias**: 鍵の名前（例: `carememo-key`）。
    *   **Validity (years)**: 25年以上を推奨します。
    *   **Certificate**: 氏名（First and Last Name）などの基本情報を入力します。
4.  入力が終わったら **OK** をクリックします。

## 2. リリース用 APK の生成
1.  **Generate Signed Bundle / APK** 画面に戻るので、作成したキーストアのパスワードとエイリアスが入力されていることを確認し、**Next** をクリックします。
2.  **Build Variant** で **release** を選択します。
3.  **Create** をクリックしてビルドを開始します。
4.  ビルドが完了すると、通知が表示されます。**locate** をクリックすると、出来上がった `app-release.apk` が表示されます。

## 3. 実機へのインストール
1.  生成された `app-release.apk` を Google ドライブや USB 経由で **KYOCERA TORQUE G06** へコピーします。
2.  スマホの「ファイル」アプリ（または Files by Google）から `app-release.apk` をタップします。
3.  「セキュリティ上の理由から、このソースからの不明なアプリをインストールすることはできません」といった警告が出た場合は、**設定** をタップし、**このソースのアプリを許可** を ON にします。
4.  **インストール** をタップして完了です。

---
## 注意事項
*   **キーストアファイル（.jks）とパスワードは絶対に紛失しないでください。** これを失うと、将来アプリを更新（上書きインストール）することができなくなります。
*   個人情報を含むバックアップデータの扱いは、リリース版でも同様に注意してください。
