# 写真記録機能 仕様書 (Ver. 1.1.0)

所見メモ（ConditionAtVisit）に対して、カメラで撮影した写真を最大3枚まで添付し、管理する機能の仕様。

## 1. ユーザー機能仕様

### 1.1. 画面構成とモード
所見詳細画面（`ConditionDetailScreen`）において、以下の2つのモードを切り替えて使用する。

- **参照モード（初期状態）**
  - 所見メモの内容（タイトル、記録者、本文）を表示。
  - 添付されている写真のサムネイルを表示。
  - サムネイルをタップすると、全画面で拡大表示（`ConditionPhotoFullScreen`）が可能。
  - 写真の追加・削除は不可。

- **編集モード（鉛筆アイコンで切替）**
  - 所見メモの内容を編集可能（音声入力対応）。
  - 写真の追加（カメラ起動）および削除が可能。
  - 記録日時の変更が可能。

### 1.2. 写真の撮影と保存
- **撮影**: Android標準のカメラ機能を呼び出して実行。
- **プレビュー**: 撮影直後に「写真の確認」画面を表示。
- **キャプション**: 撮影した写真ごとにキャプション（説明文）を登録可能。
  - デフォルト値：撮影時の日時（yyyy/MM/dd HH:mm）。
- **保存**: 「保存する」を選択した時点で、画像が最適化され内部ストレージに保存される。

### 1.3. 枚数制限
- 1つの所見メモに対して、**最大3枚**まで。
- サムネイルは横3列×縦1列で表示され、所見メモの表示領域を圧迫しないよう配慮。
- 上限に達している場合は撮影ボタンが非表示となる。

### 1.4. 履歴一覧での表示
- 所見メモ一覧（履歴）において、写真が1枚以上添付されている記録には、記録者名の左側に「カメラアイコン」を表示し、視覚的に判別可能とする。

## 2. 技術仕様

### 2.1. 保存場所とセキュリティ
- **保存先**: アプリ専用の内部ストレージ（`Context.getFilesDir()/photos/`）。
- **セキュリティ**: 他のアプリ（ギャラリーやGoogleフォト等）からはアクセスできない領域に保存し、被介護者のプライバシーを保護。
- **ライフサイクル**: アプリのアンインストール時にデータも自動削除される。

### 2.2. 画像処理
- **リサイズ**: 長辺 **1024px** に縮小し、ストレージ容量を節約。
- **サムネイル**: 表示高速化のため、長辺 **256px** のサムネイル画像を別途生成して保持。
- **回転補正**: 撮影時のExif情報に基づき、常に正しい向きで表示されるよう自動回転処理を行う。
- **形式**: JPEG形式（品質 75〜85%）で圧縮。

### 2.3. データベース構造
- **エンティティ**: `ConditionPhoto`
  - `condition_id` (外部キー: `ConditionAtVisit.id`)
  - `person_id`
  - `photo_file_name` (1024px画像名)
  - `thumbnail_file_name` (サムネイル画像名)
  - `captured_at`
  - `caption`
- **リレーション**: `ConditionAtVisit` : `ConditionPhoto` = 1 : N (N <= 3)

### 2.4. 削除挙動
- **物理削除**: 写真の削除時、および所見メモ自体の削除時は、データベースのレコードと同時に内部ストレージの画像ファイルも物理的に削除し、空き容量を維持する。

## 3. 制約事項と運用フロー

### 3.1. 新規登録時のフロー
1. 右下の「＋」ボタンより新規記録画面を開く。
2. まずテキスト情報（記録者、内容等）を入力し「保存」する。
   - ※この時点では `condition_id` が未確定のため撮影は不可。
3. 保存後、詳細画面から「編集（鉛筆）」モードに入り、写真を撮影・追加する。
   - UI上では「※所見メモの記録には、後から写真を撮影して残すこともできます」とガイドを表示。
