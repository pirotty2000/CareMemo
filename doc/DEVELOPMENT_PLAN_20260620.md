# CareMemo ブラッシュアップ計画 (2026.06.20)

スタンドアロン（1台の端末で完結）での利用を前提とした、今後の機能改善・ブラッシュアップ案。

## 1. セキュリティ：生体認証（指紋・顔認証）の導入
*   **内容**: アプリ起動時や伏せ字解除時にAndroid標準の指紋・顔認証を要求する。
*   **メリット**: 端末紛失時や盗難時のデータ流出リスクを最小化し、業務利用における機密性をさらに強固にする。

## 2. 視認性：全利用者の「注意状態」ダッシュボード
*   **内容**: メイン画面（利用者一覧）で「直近1週間に異常値（高血圧・低体重など）があった利用者」をバッジや色で強調表示する。
*   **メリット**: 管理人数が増えても、重点的なケアが必要な人を瞬時に把握でき、見落としを防ぐ。

## 3. 入力効率：数値入力のさらなる省力化
*   **内容**: 
    *   **前回値コピー**: ワンタップで前回の記録値を反映し、時刻のみを修正して保存できる機能。
    *   **直感UI**: キーボードを使わず、指のスライドだけで数値を微調整できるUIの導入。
*   **メリット**: 忙しい現場での記録時間を短縮し、片手がふさがっている状況でも正確な入力を可能にする。

## 4. 運用の継続性：バックアップのリマインド機能
*   **内容**: 最後のバックアップから一定期間（例：1週間）経過した際に、「バックアップを作成しますか？」という通知やダイアログを出す。
*   **メリット**: クラウド非連携のスタンドアロン機において、端末故障によるデータ消失リスクを最小限に抑える。

## 5. ユーザビリティ：ダークモードへの完全対応
*   **内容**: システム設定に応じた配色（ダークテーマ）への切り替え、または夜間専用モードの実装。
*   **メリット**: 夜間巡回時などの画面の眩しさを抑え、就寝中の利用者への刺激を減らしつつ、記録者の目の負担も軽減する。

## 6. 拡張性：タブレット・大画面への最適化（Adaptive UI）
*   **内容**: 画面幅に応じてレイアウトを動的に変更する。
    *   **2ペイン表示**: 左側に利用者一覧、右側に記録詳細を同時に表示する「マスター・詳細」形式の導入。
    *   **情報の並列化**: 詳細画面で「グラフ」と「履歴表」を左右に並べ、傾向と数値を同時に確認。
    *   **グリッド表示**: 利用者一覧を2〜3列のグリッド形式にして、広い画面を有効活用。
*   **メリット**: 事務作業時などの視認性と分析効率を飛躍的に向上させ、タブレットデバイスの利点を最大限に引き出す。

---
この計画は、現在の CareMemo をより「安全」「確実」「効率的」な専門職向けツールへと進化させるための指針として作成されました。
