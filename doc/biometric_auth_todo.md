# 生体認証機能の改善タスク

## 背景
生体認証に対応していない端末で「生体認証を使用する」を有効にした際、認証画面が表示されず（またはエラーになり）アプリが起動不能（画面が真っ白）になる事象が発生した。

## 目的
生体認証（指紋・顔）だけでなく、端末の画面ロック（PIN、パターン、パスワード）も認証手段として利用可能にし、可用性と利便性を向上させる。

## 実施事項

### 1. 認証方式の拡張
- `BiometricPrompt` の設定において、生体認証（`BIOMETRIC_STRONG`）だけでなくデバイス資格情報（`DEVICE_CREDENTIAL`）を許可するように変更する。
- これにより、指紋センサーがない端末でもパターンやPINでログイン可能にする。

### 2. 設定画面での事前チェック
- 「生体認証を使用する」設定を有効にする際、`BiometricManager.canAuthenticate()` を使用して、その端末で認証（生体認証またはデバイスロック）がそもそも利用可能かチェックする。
- 利用不可な場合は、スイッチをONにできないように制限し、ユーザーに警告メッセージを表示する。

### 3. 起動時の堅牢性向上
- 起動時の認証処理において、例外やエラー（`onAuthenticationError`）が発生した場合に、画面が真っ白なままフリーズしないようフォールバック処理を実装する。
- 認証がキャンセルされた場合の挙動（アプリを終了するか、別の手段を提示するか）を整理する。

### 4. テスト項目
- 生体認証（指紋・顔）が登録されている端末での正常動作。
- 生体認証はないが、画面ロック（パターン等）が設定されている端末での動作。
- 認証設定が一切されていない端末での挙動確認。
