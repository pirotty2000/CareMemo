# アプリのロック機能（生体認証・デバイス認証）の改善タスク

## 背景
生体認証に対応していない端末で「生体認証を使用する」を有効にした際、認証画面が表示されず（またはエラーになり）アプリが起動不能（画面が真っ白）になる事象が発生した。また、介護記録というデリケートな情報を扱うため、より堅牢で利便性の高いロック機能が求められている。

## 目的
- 不用意に他者に画面を見られないようにする（プライバシー保護）。
- 不用意に他者にアプリを実行されないようにする（セキュリティ向上）。
- 生体認証（指紋・顔）だけでなく、端末の画面ロック（PIN、パターン、パスワード）も認証手段として利用可能にし、可用性と利便性を向上させる。

## 実施事項

### 1. 認証方式の拡張
- `BiometricPrompt` の設定において、生体認証（`BIOMETRIC_STRONG`）だけでなくデバイス資格情報（`DEVICE_CREDENTIAL`）を許可するように変更する。
- これにより、指紋センサーがない端末でもパターンやPINでログイン可能にする。
- 「生体認証」という表記を「アプリのロック」や「画面ロックで認証」といった、より広い意味の表現に変更を検討する。

### 2. 設定画面での事前チェック
- ロック設定を有効にする際、`BiometricManager.canAuthenticate()` を使用して、その端末で認証（生体認証またはデバイスロック）がそもそも利用可能かチェックする。
- 利用不可な場合は、スイッチをONにできないように制限（グレーアウト）し、ユーザーに「端末の設定で画面ロックを有効にしてください」等のメッセージを表示する。

### 3. 起動時および復帰時の認証ロジック
- **起動時の堅牢性向上:** 認証処理において例外やエラー（`onAuthenticationError`）が発生した場合に、画面が真っ白なままフリーズしないようフォールバック処理（再試行ボタンの表示や、最悪の場合のアプリ終了処理など）を実装する。
- **自動ロック機能（タイムアウト）の導入:**
    - アプリがバックグラウンドに回ってから、一定時間経過した後に再度認証を求める機能を実装する。
    - 設定項目として「再ロックまでの時間（即時、1分、5分、30分など）」を追加する。
    - `ProcessLifecycleOwner` を使用してアプリのフォアグラウンド/バックグラウンド遷移を監視し、経過時間を判定する。

### 4. プライバシー保護（画面隠匿）
- `Window.addFlags(WindowManager.LayoutParams.FLAG_SECURE)` を検討する。
- これにより、OSの「履歴画面（タスクスウィッチャー）」でアプリの内容がプレビュー表示されるのを防ぎ、スクリーンショットも制限する。

### 5. テスト項目
- 生体認証（指紋・顔）が登録されている端末での正常動作。
- 生体認証はないが、画面ロック（パターン等）が設定されている端末での動作。
- 認証設定が一切されていない端末での挙動確認（設定変更が阻止されるか）。
- 指定した時間経過後に正しく再ロックがかかるかの確認。
- 履歴画面で内容が隠蔽されているかの確認。
