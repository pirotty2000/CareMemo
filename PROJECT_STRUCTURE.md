# CareMemo（健康管理メモ帳）プロジェクト構成

このドキュメントは、プロジェクトの学習用として各ファイルの役割と構成をまとめたものです。

## 1. プロジェクト概要
利用者の身体状況（身長・体重、血圧、血糖値、所見など）を記録・管理するための Android アプリケーションです。
Jetpack Compose (UI) と Room (Database) を組み合わせた、現代的な Android アプリの構成をとっています。

## 2. ディレクトリ構成と役割

### 📂 data（データ層）
データベースに関連する定義と、データの永続化を担当します。

*   **`Entity.kt`**: 
    *   データベースのテーブル定義（利用者の基本情報や各記録項目）。
    *   論理削除用のフラグや、メイン画面のバッジ表示用サマリー構造体も含みます。
*   **`Dao.kt`**: 
    *   Database Access Object。SQL クエリの定義場所です。
    *   データの挿入、更新、検索、削除（物理・論理）を実行します。
*   **`CareMemoRepository.kt`**: 
    *   ViewModel とデータソース（Dao）の仲介役。
    *   「利用者を削除したらその記録も一括で削除する」といったビジネスロジックの橋渡しを行います。
*   **`Category.kt`**: 
    *   アプリで扱う 4 つの記録カテゴリー（身・圧・糖・メ）の定義。
*   **`AppDatabase.kt`**:
    *   Room データベースの本体設定。

### 📂 viewmodel（ロジック層）
UI とデータの橋渡しを行い、画面の状態（State）を管理します。

*   **`PersonListViewModel.kt`**: 
    *   メイン画面（利用者一覧）用のロジック。
    *   利用者の検索、バッジ点灯情報の統合計算、データのインポート/エクスポートを担当。
*   **`PersonDetailViewModel.kt`**: 
    *   詳細画面（記録入力・グラフ）用のロジック。
    *   選択された利用者の各記録の読み込み、保存、削除を担当。

### 📂 ui（表示層）
Jetpack Compose を使用したユーザーインターフェースです。

#### 📂 screens（各画面）
*   **`MainScreen.kt`**: 
    *   アプリの顔となる「利用者一覧」画面。
*   **`UnifiedRecordScreen.kt`**: 
    *   このアプリで最も複雑かつ重要な画面。
    *   1 つの画面で 4 つのカテゴリーの「入力」「履歴表示」「グラフ表示」を切り替えて行います。
*   **`DeletedUserListScreen.kt`**: 
    *   「利用終了」とした人を一覧表示し、復帰させるための画面。

#### 📂 components（共通部品）
*   **`CategoryBadges.kt`**: 
    *   一覧画面に表示される「身・圧・糖・メ」の 2×2 バッジ表示用コンポーネント。

#### 📂 theme（デザイン）
*   **`Color.kt` / `Theme.kt`**: 
    *   アプリ全体の色調やフォントなどのスタイル定義。

## 3. アプリの主要な仕組み（学習ポイント）

1.  **論理削除 (Logical Delete)**:
    *   データを物理的に消すのではなく、`deleted_at` タイムスタンプを入れることで「非表示」として扱います。これにより「元に戻す」操作を実現しています。
2.  **リアルタイムな状態監視 (Flow)**:
    *   `combine` 関数を使い、データベース内の複数のテーブル（血圧、血糖など）を同時に監視して、一覧画面のバッジを自動的に点灯させています。
3.  **時刻の正規化**:
    *   ユーザーの要望により「秒」を排除し、分単位までの管理とすることで入力の手間を軽減しています。
