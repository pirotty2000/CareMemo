# 保存機能の仕様書

本ドキュメントは、CareMemo アプリにおけるデータ保存（バリデーション、一意性担保、重複防止）の仕様と実装状況を管理するものです。

## 1. 全体方針
- **データの堅牢性**: データベースレベルの制約（Unique Index）を活用し、意図しないデータの重複を防ぐ。
- **時間の精度**: 記録データの一意性判断をユーザーの認識（分単位）に合わせるため、秒以下を切り捨てて保存する。
- **入力の柔軟性**: 記録データにおいては、知り得た情報のみ（一部空欄でも可）での保存を許容する。

## 2. 利用者（Person）の登録・更新
- **一意性制約**: `氏名` + `生年月日` + `同姓同名識別用メモ` の組み合わせを一意とする。
- **重複発生時の挙動**: 
  - 保存を中断し、エラーダイアログを表示する。
  - ユーザーに対し「識別用メモ」の入力を促し、別データとして登録するよう誘導する。
- **識別用メモ**: 
  - デフォルト値は空文字 (`""`)。
  - 同姓同名の利用者を安全に管理するための任意項目。

## 3. 詳細データ（記録）の保存
- **対象**: `HeightAndWeight`, `BpAndPulse`, `GlucoseAndHbA1c`, `ConditionAtVisit`
- **一意性制約**: `利用者ID` + `記録日時（分単位）` の組み合わせを一意とする。
- **時間の丸め込み**: 
  - 保存直前にタイムスタンプを **分単位で切り捨て (`truncatedTo(ChronoUnit.MINUTES)`)** 処理する。
  - これにより、同じ「分」に保存されたデータは同一とみなされる。
- **重複発生時の挙動**: 
  - **自動上書き (REPLACE)**。
  - 連打による二重登録を防止しつつ、微修正をスムーズに行えるようにする。
- **バリデーション**:
  - 全入力項目が未入力（日時以外）の場合は「保存するデータがありません」と Snackbar で通知し、保存を拒否する。
  - 一部でも項目が埋まっていれば保存可能。
